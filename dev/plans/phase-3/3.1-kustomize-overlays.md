# 3.1 Kustomize Overlays

## Context

Current deployment uses raw numbered YAML manifests in `strimzi-mcp/install/`. For GitOps workflows (ArgoCD, Flux) and environment-specific customization, we need Kustomize base + overlays. The project already uses Kustomize in `dev/manifests/` for Strimzi/Prometheus/Loki setup.

## What Changes

### New Directory Structure

```
strimzi-mcp/install/
├── base/
│   ├── kustomization.yaml
│   ├── namespace.yaml          (from 001-Namespace.yaml)
│   ├── serviceaccount.yaml     (from 002-ServiceAccount.yaml)
│   ├── clusterrole.yaml        (from 003-ClusterRole.yaml)
│   ├── clusterrolebinding.yaml (from 004-ClusterRoleBinding.yaml)
│   ├── deployment.yaml         (from 005-Deployment.yaml)
│   ├── service.yaml            (from 006-Service.yaml)
│   └── role-sensitive.yaml     (from 007-Role-sensitive.yaml)
├── overlays/
│   ├── dev/
│   │   ├── kustomization.yaml
│   │   └── deployment-patch.yaml  (1 replica, no auth, imagePullPolicy: IfNotPresent)
│   ├── prod/
│   │   ├── kustomization.yaml
│   │   ├── deployment-patch.yaml  (replicas, resource limits, auth env vars)
│   │   ├── clusterrole-impersonation-patch.yaml  (adds impersonation verbs)
│   │   └── auth-config.yaml  (OIDC environment variables)
│   └── prod-openshift/
│       ├── kustomization.yaml
│       ├── deployment-patch.yaml
│       └── route.yaml  (OpenShift Route for external access)
└── README.md  (migration guide from old numbered files)
```

### Base kustomization.yaml

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - namespace.yaml
  - serviceaccount.yaml
  - clusterrole.yaml
  - clusterrolebinding.yaml
  - deployment.yaml
  - service.yaml
commonLabels:
  app.kubernetes.io/name: strimzi-mcp
  app.kubernetes.io/part-of: streamshub
```

### Overlay Features

**Dev overlay:**
- 1 replica, relaxed resources
- OIDC disabled
- `imagePullPolicy: IfNotPresent` for local builds

**Prod overlay:**
- Configurable replicas via patches
- OIDC env vars (server URL, client ID)
- Impersonation RBAC added to ClusterRole
- Tighter resource limits
- Namespace filtering via `MCP_NAMESPACES` env var

**Prod-OpenShift overlay:**
- Extends prod overlay
- Adds OpenShift Route

### Migration

- Old numbered files remain temporarily with deprecation notice
- README documents migration: `kubectl apply -k install/overlays/prod/` instead of `kubectl apply -f install/`

## Key Notes

- Sensitive Role (`role-sensitive.yaml`) stays separate — users include it per-namespace as needed
- RoleBinding for sensitive role is user-created (namespace-specific, can't be templated generically)
- Namespace in base is `streamshub-mcp` — overlays can override via `namespace:` field in kustomization.yaml

## Verification

1. `kubectl apply -k install/base/ --dry-run=client` — validates base
2. `kubectl apply -k install/overlays/dev/ --dry-run=client` — validates dev overlay
3. `kubectl apply -k install/overlays/prod/ --dry-run=client` — validates prod overlay
4. Deploy dev overlay on Kind cluster — MCP server starts, tools work
5. Deploy prod overlay with Keycloak — auth enforced
