# 3.4 JaCoCo Code Coverage

## Context

Tests exist but coverage isn't measured or enforced. JaCoCo provides coverage reports and can enforce minimums in CI.

## What Changes

### Modified Files

**`pom.xml` (parent)** — add to `<pluginManagement>`:
```xml
<plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <version>0.8.12</version>
</plugin>
```

**Each module `pom.xml`** (common, strimzi-mcp, metrics-prometheus, loki-log-provider) — add:
```xml
<plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <executions>
        <execution>
            <goals><goal>prepare-agent</goal></goals>
        </execution>
        <execution>
            <id>report</id>
            <phase>test</phase>
            <goals><goal>report</goal></goals>
        </execution>
    </executions>
</plugin>
```

**`.github/workflows/build.yml`** — add coverage report upload step (optional: Codecov or GitHub Actions artifact)

### No Minimum Enforcement Initially

Start with reporting only. Add enforcement (`check` goal with minimum thresholds) after baseline is established.

## Verification

1. `mvn clean test` — JaCoCo agent runs, generates `target/site/jacoco/` reports
2. Review coverage percentages for each module
3. CI uploads coverage report as artifact
