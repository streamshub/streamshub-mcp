# 1.1 OIDC Authentication

## Context

The MCP HTTP endpoint (`/mcp/*`) is completely open — no authentication at all. The auth architecture doc specifies Keycloak PAT/OIDC as the primary auth mechanism. We use Quarkus built-in HTTP security (`quarkus-oidc`) to validate tokens at the HTTP layer before any MCP dispatch occurs. Auth is optional: disabled in dev profile, enabled in prod.

## What Changes

### Dependencies

**`common/pom.xml`** — add:
```xml
<dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-oidc</artifactId>
</dependency>
```

### New Files

**`common/src/main/java/io/streamshub/mcp/common/security/McpSecurityContext.java`**
- `@RequestScoped` CDI bean
- Injects `io.quarkus.security.identity.SecurityIdentity`
- Exposes: `getUserId()` (from `sub` claim), `getEmail()` (from `email` claim), `getGroups()` (from `groups` claim)
- When auth is disabled (dev), returns anonymous identity with sensible defaults
- Services and filters inject this to get the current user

**`common/src/main/java/io/streamshub/mcp/common/security/McpIdentity.java`**
- Immutable record: `McpIdentity(String userId, String email, Set<String> groups)`
- `McpSecurityContext.getIdentity()` returns this
- Static `ANONYMOUS` constant for unauthenticated mode

### Modified Files

**`strimzi-mcp/src/main/resources/application.properties`** — add:
```properties
# OIDC Authentication (disabled by default, enable in prod)
quarkus.oidc.enabled=false
%prod.quarkus.oidc.enabled=true
quarkus.oidc.auth-server-url=https://keycloak.example.com/realms/firma
quarkus.oidc.client-id=strimzi-mcp
quarkus.oidc.token.audience=strimzi-mcp

# HTTP security policies
quarkus.http.auth.permission.mcp.paths=/mcp/*
quarkus.http.auth.permission.mcp.policy=authenticated
quarkus.http.auth.permission.mcp.enabled=${quarkus.oidc.enabled}
quarkus.http.auth.permission.health.paths=/q/*
quarkus.http.auth.permission.health.policy=permit

# CORS - add Authorization header
quarkus.http.cors.headers=Accept,Content-Type,Mcp-Session-Id,Authorization
```

**`strimzi-mcp/install/005-Deployment.yaml`** — add env vars:
```yaml
- name: QUARKUS_OIDC_AUTH_SERVER_URL
  value: ""  # User must configure
- name: QUARKUS_OIDC_CLIENT_ID
  value: "strimzi-mcp"
```

### Test Files

**`common/src/test/java/io/streamshub/mcp/common/security/McpSecurityContextTest.java`**
- Test with mocked `SecurityIdentity` — verify claim extraction
- Test anonymous mode (when OIDC disabled) — verify `ANONYMOUS` identity returned
- Test missing claims — verify graceful handling (null email, empty groups)

**`strimzi-mcp/src/test/java/io/streamshub/mcp/strimzi/OidcAuthenticationTest.java`**
- `@QuarkusTest` with `@TestProfile` enabling OIDC
- Use `@OidcSecurity` test annotations for mock OIDC tokens
- Test: authenticated request succeeds
- Test: unauthenticated request returns 401
- Test: expired token returns 401
- Test: health endpoints accessible without auth

## Key Patterns to Follow

- `McpSecurityContext` follows the same `@RequestScoped` pattern as Quarkus `SecurityIdentity` itself
- Configuration uses Quarkus profile-based gating (`%prod.` prefix) — same pattern as existing `%dev.` configs
- No custom token validation logic — delegate entirely to `quarkus-oidc`
- CORS header list already exists in `application.properties` line ~5 — just add `Authorization`

## Verification

1. `mvn clean test` — all existing tests pass (OIDC disabled by default)
2. `mvn quarkus:dev` — dev mode works without Keycloak
3. Run with `QUARKUS_OIDC_ENABLED=true` and a Keycloak instance — `/mcp` requires Bearer token, `/q/health` open
4. New unit tests pass for `McpSecurityContext`
