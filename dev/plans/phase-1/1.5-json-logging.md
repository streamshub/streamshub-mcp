# 1.5 JSON Logging for Production

## Context

Current logging uses console format only (`%d{HH:mm:ss} %-5p [%c{2.}] (%t) %s%e%n`). Production deployments need JSON-formatted logs for ingestion by ELK, CloudWatch, Loki, or similar systems.

## What Changes

### Modified Files

**`strimzi-mcp/pom.xml`** — add dependency:
```xml
<dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-logging-json</artifactId>
</dependency>
```

**`strimzi-mcp/src/main/resources/application.properties`** — add:
```properties
# JSON logging (enabled in prod, disabled in dev for readability)
quarkus.log.console.json=false
%prod.quarkus.log.console.json=true
quarkus.log.console.json.additional-field."service".value=strimzi-mcp
quarkus.log.console.json.additional-field."service".type=string
```

### No New Files

This is a dependency + configuration change only.

### Test Verification

No new test files needed — this is a logging format change. Verify manually:
1. `mvn quarkus:dev` — console format preserved in dev mode
2. Run with `QUARKUS_LOG_CONSOLE_JSON=true` — verify JSON output
3. Verify audit logs (1.3) render as proper JSON objects within the JSON log stream

## Notes

- `quarkus-logging-json` is a standard Quarkus extension, zero code required
- The `service` additional field helps filter MCP server logs in multi-service deployments
- Exception stack traces are included as JSON arrays (not raw multiline text)
- Structured fields from JBoss Logger `infof()` calls are preserved in JSON output
