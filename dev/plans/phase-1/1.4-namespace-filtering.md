# 1.4 Namespace Filtering

## Context

All resource queries currently use `.inAnyNamespace()` and `ResourceSubscriptionManager` watches everything. The proposal requires `MCP_NAMESPACES` (or `mcp.namespaces`) to restrict which namespaces the server sees. In production clusters with hundreds of namespaces, this causes excessive K8s API load and exposes resources beyond the operator's scope.

## What Changes

### New Files

**`common/src/main/java/io/streamshub/mcp/common/config/NamespaceConfig.java`**
- `@ConfigMapping(prefix = "mcp")`
- `Optional<List<String>> namespaces()` â€” comma-separated list of namespaces (empty = all)
- Validates namespace names against K8s conventions on startup

**`common/src/main/java/io/streamshub/mcp/common/service/NamespaceFilter.java`**
- `@ApplicationScoped` utility bean
- `isAllNamespaces()` â€” true when no filter configured
- `getAllowedNamespaces()` â€” returns configured set
- `isAllowed(String namespace)` â€” check if namespace is in allowed set
- `<T> FilteredOperation<T> applyToList(MixedOperation<T,...>)` â€” returns `.inAnyNamespace()` or `.inNamespace(ns)` for each configured namespace, merging results
- Used by `KubernetesResourceService` and all services that query K8s

### Modified Files

**`common/src/main/java/io/streamshub/mcp/common/service/KubernetesResourceService.java`**
- Inject `NamespaceFilter`
- Replace all `.inAnyNamespace()` calls with namespace-filtered equivalents
- When `isAllNamespaces()`, behavior unchanged
- When namespaces configured, issue per-namespace queries and merge results

**`strimzi-mcp/src/main/java/io/streamshub/mcp/strimzi/resource/ResourceSubscriptionManager.java`**
- Inject `NamespaceFilter`
- When namespaces configured, create per-namespace watches instead of cluster-wide watches
- Each namespace gets its own informer/watcher

**`common/src/main/java/io/streamshub/mcp/common/service/CompletionHelper.java`**
- Namespace completion results filtered through `NamespaceFilter`
- Only suggest namespaces that are in the allowed set

**`strimzi-mcp/src/main/java/io/streamshub/mcp/strimzi/service/CompletionService.java`**
- Resource completions (cluster names, topic names, etc.) already go through services that will be filtered
- Verify no direct K8s calls bypass `NamespaceFilter`

## Key Design Decisions

- **Per-namespace queries vs. cluster-wide + client-side filter**: Per-namespace queries are preferred because they respect K8s RBAC (the SA might only have namespaced RoleBindings) and reduce API server load. Client-side filtering would still fetch all resources.
- **Environment variable**: `MCP_NAMESPACES` (Quarkus auto-maps from `mcp.namespaces`) â€” matches proposal specification.
- **Watch strategy**: Multiple per-namespace watches vs. single cluster-wide watch with filter. Per-namespace watches are more efficient and RBAC-compatible.
