# 2.6 DrainCleaner Tools

## Context

Strimzi Drain Cleaner is a must-have for production Kafka on Kubernetes. It handles graceful pod evictions during node drains/maintenance by intercepting eviction requests via a ValidatingWebhookConfiguration and annotating pods so Strimzi can perform proper rolling updates instead of abrupt evictions. Without it, node maintenance can cause data loss or broker unavailability.

DrainCleaner is deployed as a separate Deployment (not a CRD) with a webhook configuration. It's in a separate repository (`strimzi/drain-cleaner`) but is a critical part of the Strimzi ecosystem.

## New Tools

### `list_drain_cleaners`
- **Parameters:** `namespace` (optional)
- **Returns:** List of DrainCleaner deployments: name, namespace, replicas (expected/ready), image, status, uptime
- **Discovery:** Look for Deployments with label `app.kubernetes.io/name=strimzi-drain-cleaner` or `app=strimzi-drain-cleaner`

### `get_drain_cleaner`
- **Parameters:** `namespace` (optional), `name` (required)
- **Returns:** Detailed deployment: image, replicas, resources, pod status, webhook configuration (name, failurePolicy, namespaceSelector, rules), TLS cert status
- **Additional queries:**
  - Read the `ValidatingWebhookConfiguration` to check webhook is properly configured
  - Check if the webhook cert/secret exists and is valid
  - Report failurePolicy (should be `Ignore` for safety or `Fail` for strict mode)

### `check_drain_cleaner_readiness`
- **Parameters:** `namespace` (optional)
- **Returns:** Production readiness assessment:
  - Is DrainCleaner deployed? (critical check)
  - Are all replicas ready?
  - Is the webhook configured correctly?
  - What's the failure policy? (`Fail` = strict, `Ignore` = permissive)
  - Are certificates valid and not expiring soon?
  - Which namespaces are covered by the webhook selector?
- **Use case:** Pre-production checklist, upgrade readiness

## RBAC

Add to `003-ClusterRole.yaml`:
```yaml
- apiGroups: ["admissionregistration.k8s.io"]
  resources: ["validatingwebhookconfigurations"]
  verbs: ["get", "list"]
```
Deployments are already covered by existing RBAC.

## Key Notes

- DrainCleaner is NOT a CRD ŌĆö it's a standard Deployment + ValidatingWebhookConfiguration
- Detection is label-based since there's no custom resource
- Webhook cert is stored in a Secret ŌĆö only check expiry, never expose cert data (same pattern as `KafkaCertificateService`)
- If DrainCleaner is not found, report this clearly ŌĆö it's a production concern
