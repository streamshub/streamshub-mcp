# 2.1 KafkaConnect + KafkaConnector Tools

## Context

KafkaConnect and KafkaConnector are the most commonly troubleshot Strimzi CRDs in production. Customers constantly ask "why is my connector failing?" and "what's the connector status?" These are managed by the Strimzi Cluster Operator and expose rich status information. This is the single highest-value addition to the MCP.

## New Tools

### `list_kafka_connects`
- **Parameters:** `namespace` (optional)
- **Returns:** List of KafkaConnect clusters with: name, namespace, replicas (expected/ready), image, status conditions, build status
- **Service:** `KafkaConnectService.listConnects(namespace)`
- **K8s CRD:** `kafkaconnects.kafka.strimzi.io`

### `get_kafka_connect`
- **Parameters:** `namespace` (optional), `connectName` (required)
- **Returns:** Detailed Connect cluster: bootstrapServers, replicas, image, config (group.id, key/value converters), build spec, authentication type, TLS config, status conditions
- **Service:** `KafkaConnectService.getConnect(namespace, name)`

### `list_kafka_connectors`
- **Parameters:** `namespace` (optional), `connectCluster` (optional — filter by parent Connect cluster)
- **Returns:** List of connectors with: name, class, tasksMax, state (running/paused), auto-restart config, topics, status conditions
- **Service:** `KafkaConnectorService.listConnectors(namespace, connectCluster)`
- **K8s CRD:** `kafkaconnectors.kafka.strimzi.io`
- **Label filter:** When `connectCluster` specified, filter by `strimzi.io/cluster` label

### `get_kafka_connector`
- **Parameters:** `namespace` (optional), `connectorName` (required)
- **Returns:** Detailed connector: class, tasksMax, config map, state, auto-restart status (count, last restart time), connector status from Connect REST API (as reported in CR status), topics list
- **Service:** `KafkaConnectorService.getConnector(namespace, name)`

### `get_kafka_connect_logs`
- **Parameters:** `namespace` (optional), `connectName` (required), `filter` (optional), `keywords` (optional), `sinceMinutes`/`startTime`/`endTime` (optional), `tailLines` (optional)
- **Returns:** Aggregated logs from Connect pods with error analysis
- **Service:** Reuse `LogCollectionService` from common module — same pattern as `get_kafka_cluster_logs`
- **Pod selection:** Pods with `strimzi.io/cluster={connectName}` and `strimzi.io/kind=KafkaConnect`

### `get_kafka_connect_pods`
- **Parameters:** `namespace` (optional), `connectName` (required)
- **Returns:** Pod summaries: name, phase, readiness, restarts, age, resources
- **Service:** Reuse `PodsService` — same pattern as `get_kafka_cluster_pods`

### `diagnose_kafka_connector`
- **Composite diagnostic tool** following existing diagnostic pattern
- **Steps:**
  1. Get connector status and conditions
  2. Get parent KafkaConnect cluster status
  3. Get Connect pod health (restarts, readiness)
  4. Get Connect logs filtered for connector class name
  5. Get Strimzi events for Connect and connector CRs
  6. (If Sampling supported) LLM triage of collected data
  7. (If Sampling supported) Root cause analysis
- **Service:** `KafkaConnectorDiagnosticService` — follows `KafkaClusterDiagnosticService` pattern

## New Prompt

### `troubleshoot-connector`
- **Parameters:** `connector_name`, `namespace` (optional), `connect_cluster` (optional), `symptom` (optional)
- **Workflow:**
  1. Get connector status → check for Failed state, task failures
  2. Get KafkaConnect cluster status → check readiness
  3. Get Connect pods → check for crashes, restarts
  4. Get Connect logs with connector class as keyword
  5. Get events for connector and Connect CRs
  6. Correlate: distinguish connector config errors from Connect platform issues
- **File:** `TroubleshootConnectorPrompt.java`

## RBAC

No changes to `003-ClusterRole.yaml` needed — `kafkaconnects` and `kafkaconnectors` are in the `kafka.strimzi.io` API group which is already permitted with `get`, `list`, `watch`.

## Key Patterns to Follow

- **Service layer:** Extract fields from CRDs using Strimzi API types (`KafkaConnect`, `KafkaConnector`, `KafkaConnectSpec`, etc.)
- **Tool layer:** Thin wrappers that call service methods, use `InputUtils.normalizeInput()`, throw `ToolCallException` for validation errors
- **Diagnostic:** Follow `KafkaClusterDiagnosticService` — use `DiagnosticHelper`, `Sampling`, `Elicitation`, progress callbacks
- **Logs:** Reuse `LogCollectionService` with appropriate label selectors
- **Constants:** Add Connect-specific constants to `StrimziConstants.java` (labels, component types, URIs)
