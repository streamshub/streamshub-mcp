# 4.3 Tool Argument Completions

## Context

Diagnostic tools and new CRD tools lack argument completion — users must type cluster names, namespaces, connector names, etc. manually. MCP supports tool argument completions via `@CompleteToolArg` (depends on Quarkus MCP Server API support).

## What Changes

### New File

**`strimzi-mcp/src/main/java/io/streamshub/mcp/strimzi/tool/ToolCompletions.java`**
- `@CompleteToolArg` handlers for tools that take resource names as arguments:
  - `clusterName` → query `list_kafka_clusters` for cluster names
  - `namespace` → query K8s namespaces (reuse `CompletionHelper`)
  - `connectName` → query KafkaConnect resources
  - `connectorName` → query KafkaConnector resources
  - `userName` → query KafkaUser resources
  - `rebalanceName` → query KafkaRebalance resources
  - `mirrorMakerName` → query KafkaMirrorMaker2 resources
  - `bridgeName` → query KafkaBridge resources
  - `topicName` → query KafkaTopic resources

### Modified Files

- Reuse `CompletionService` patterns from existing prompt completions
- Update `McpDiscoveryTest.java` to verify tool completions registered

### Dependency

- Requires Quarkus MCP Server to support `@CompleteToolArg` API — verify this exists in version 1.11.1+
- If not available, this item is deferred until the API is available

## Verification

1. `mvn clean test` — completion tests pass
2. Use MCP client that supports tool arg completions — verify suggestions appear
