# 4.2 Additional Prompts

## Context

Two more prompt templates to complete the expert workflow coverage: topic troubleshooting and capacity analysis.

## New Prompts

### `troubleshoot-topic`
- **Parameters:** `topic_name`, `cluster_name` (optional), `namespace` (optional), `symptom` (optional)
- **Workflow:**
  1. Get topic status and conditions → check for NotReady, stalled reconciliation
  2. List node pools → compare broker count vs replication factor
  3. Get cluster logs filtered by topic name keyword
  4. Get events for topic-related occurrences
  5. If symptom mentions "lag" → get Kafka Exporter metrics for consumer group lag on this topic
  6. Correlate: config mismatches, replication issues, Topic Operator errors

### `analyze-capacity`
- **Parameters:** `cluster_name`, `namespace` (optional), `concern` (optional)
- **Workflow:**
  1. Get cluster status → replicas, version
  2. Get node pool details → resources (CPU/memory requests/limits), storage
  3. Get metrics: throughput (bytes in/out), resource utilization (heap, CPU)
  4. Get metrics: partition count, replication factor
  5. Assess: resource headroom, storage utilization, throughput trends relative to capacity
  6. Recommend: scale triggers, resource adjustments
