# 5.3 E2E Auth with Keycloak

## Context

OIDC auth and K8s impersonation (Phase 1) need E2E verification with a real Keycloak instance on Kind.

## Test Infrastructure

### Keycloak Deployment on Kind

Add Keycloak to E2E setup:
- Deploy Keycloak operator or standalone Keycloak on Kind
- Create realm `test-realm` with:
  - Client `strimzi-mcp` (confidential, service account enabled)
  - User `user-a` in groups `["ns-kafka-a-rw"]`
  - User `user-b` in groups `["ns-kafka-b-rw"]`
  - User `admin` in groups `["platform-admin"]`
- Create namespace-scoped RoleBindings:
  - `ns-kafka-a-rw` group → `edit` role in `kafka-a` namespace
  - `ns-kafka-b-rw` group → `edit` role in `kafka-b` namespace
  - `platform-admin` group → `cluster-admin`

### MCP Server Configuration

Deploy MCP with:
```properties
QUARKUS_OIDC_ENABLED=true
QUARKUS_OIDC_AUTH_SERVER_URL=http://keycloak:8080/realms/test-realm
MCP_SECURITY_IMPERSONATION_ENABLED=true
```

## Test Classes

### `OidcAuthE2ETest.java`

**Test: Unauthenticated request rejected**
- Call `/mcp` without Bearer token → expect 401

**Test: Invalid token rejected**
- Call `/mcp` with garbage Bearer token → expect 401

**Test: Expired token rejected**
- Generate token with very short expiry, wait, then call → expect 401

**Test: Valid token accepted**
- Get token for `user-a` → call tool → expect success

**Test: Health endpoint accessible without auth**
- Call `/q/health` without token → expect 200

### `ImpersonationE2ETest.java`

**Test: User A sees namespace A only**
- Auth as `user-a` → `list_kafka_clusters` → should only see clusters in `kafka-a` namespace
- K8s RBAC enforces this via impersonation

**Test: User B sees namespace B only**
- Auth as `user-b` → `list_kafka_clusters` → should only see clusters in `kafka-b` namespace

**Test: Admin sees all namespaces**
- Auth as `admin` → `list_kafka_clusters` → sees all clusters

**Test: Cross-namespace access denied**
- Auth as `user-a` → `get_kafka_cluster` in `kafka-b` namespace → expect 403/error

### `AuditLoggingE2ETest.java`

**Test: Tool calls produce audit log entries**
- Auth as `user-a` → call tool → check server logs for audit entry with `userId=user-a`
- Verify: timestamp, tool name, arguments, result, duration present

## Key Considerations

- Keycloak startup takes 30-60 seconds — factor into CI timing
- Consider using Keycloak testcontainer or lightweight OIDC mock for faster tests
- K8s on Kind needs OIDC configured (or use impersonation which doesn't require K8s OIDC config)
- MCP ServiceAccount needs impersonation RBAC

## Verification

1. All auth E2E tests pass
2. Namespace isolation is properly enforced
3. Audit logs contain correct user identity
