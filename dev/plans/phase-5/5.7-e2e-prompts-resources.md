# 5.7 E2E Prompts & Resources

## Context

Verify all prompts return valid workflow instructions, all resource templates return current state, subscriptions deliver notifications, and completions return valid suggestions.

## Test Classes

### `PromptE2ETest.java`

**Test each prompt returns valid instructions:**
- `diagnose-cluster-issue` → returns PromptMessages referencing existing tool names
- `troubleshoot-connectivity` → returns valid workflow
- `analyze-kafka-metrics` → returns valid workflow
- `analyze-strimzi-operator-metrics` → returns valid workflow
- `troubleshoot-connector` → returns valid workflow (Phase 2)
- `audit-security` → returns valid workflow (Phase 2)
- `assess-upgrade-readiness` → returns valid workflow (Phase 2)
- `troubleshoot-topic` → returns valid workflow (Phase 4)
- `analyze-capacity` → returns valid workflow (Phase 4)

**Test prompt with parameters:**
- Pass cluster_name, namespace, symptom → verify they're injected into instructions

### `ResourceTemplateE2ETest.java`

**Test each resource returns valid JSON:**
- `strimzi://kafka.strimzi.io/namespaces/{ns}/kafkas/{name}/status` → valid Kafka status JSON
- `strimzi://kafka.strimzi.io/namespaces/{ns}/kafkas/{name}/topology` → valid topology JSON
- All 5 existing + 6 new resource template URIs
- Verify non-existent resource returns appropriate error

### `ResourceSubscriptionE2ETest.java`

**Test subscription delivers update notifications:**
- Subscribe to Kafka cluster status resource
- Modify Kafka CR (e.g., add an annotation to trigger reconciliation)
- Wait for notification → verify it arrives within reasonable time
- Verify deduplication: if nothing changes, no duplicate notification

### `CompletionE2ETest.java`

**Test prompt completions return valid suggestions:**
- Complete `cluster_name` parameter → get list of actual cluster names
- Complete `namespace` parameter → get list of actual namespaces
- Complete with partial prefix → verify filtering works

**Test resource template completions:**
- Complete `{namespace}` → get actual namespaces
- Complete `{name}` for Kafka → get actual cluster names

## Verification

1. All prompts, resources, subscriptions, and completions work against real cluster
2. Data returned matches actual cluster state
3. Subscriptions are not flaky (use appropriate wait conditions)
