# 5.5 E2E Metrics & Logging

## Context

Verify server self-monitoring metrics (Phase 3.2) are scraped correctly, JSON logging works in prod profile, and metrics tools auto-detect Metrics Reporter vs JMX Exporter.

## Test Classes

### `ServerMetricsE2ETest.java`

**Test: Prometheus metrics endpoint accessible**
- `curl /q/metrics` → 200 with Prometheus text format

**Test: Tool call metrics recorded**
- Invoke `list_kafka_clusters` 3 times
- Scrape `/q/metrics` → verify `mcp_tool_calls_total{tool="list_kafka_clusters"}` = 3
- Verify `mcp_tool_call_duration_seconds` histogram has entries

**Test: Guardrail metrics recorded**
- Trigger rate limit → verify `mcp_guardrail_rate_limit_rejected_total` incremented
- Invoke tool with large response → verify truncation metric if triggered

### `JsonLoggingE2ETest.java`

**Test: JSON log output in prod profile**
- Deploy MCP with prod profile
- Invoke tool → capture pod logs
- Verify each log line is valid JSON with expected fields (timestamp, level, message, loggerName)

**Test: Audit logs in JSON**
- Invoke tool → capture logs → find audit entry → verify it's structured JSON

### `MetricsReporterDetectionE2ETest.java`

**Test: Auto-detect Metrics Reporter**
- Deploy Kafka cluster with Metrics Reporter (metricsConfig type = kafkaMetricsReporter)
- `get_kafka_metrics` → verify metrics collected from port 8080

**Test: Auto-detect JMX Exporter**
- Deploy Kafka cluster with JMX Exporter (metricsConfig type = jmxPrometheusExporter)
- `get_kafka_metrics` → verify metrics collected from port 9404

**Test: Fallback when no metrics configured**
- Deploy Kafka cluster without metricsConfig
- `get_kafka_metrics` → verify clear error message

## Verification

1. All metrics and logging tests pass
2. Prometheus scraping of MCP server works in CI (Prometheus already deployed for E2E)
