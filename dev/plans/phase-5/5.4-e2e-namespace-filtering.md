# 5.4 E2E Namespace Filtering

## Context

Namespace filtering (Phase 1.4) restricts which namespaces the MCP server queries. E2E tests verify this works across all tool types, watches, and completions.

## Test Setup

- Deploy Kafka clusters in 3 namespaces: `ns-allowed-1`, `ns-allowed-2`, `ns-excluded`
- Configure MCP with `MCP_NAMESPACES=ns-allowed-1,ns-allowed-2`

## Test Classes

### `NamespaceFilteringE2ETest.java`

**Test: list tools respect filter**
- `list_kafka_clusters` → returns clusters from `ns-allowed-1` and `ns-allowed-2` only
- Cluster in `ns-excluded` not visible

**Test: get tool with excluded namespace**
- `get_kafka_cluster(namespace=ns-excluded, name=...)` → not found or access denied

**Test: watches scoped to configured namespaces**
- Subscribe to Kafka resource updates
- Modify cluster in `ns-allowed-1` → notification received
- Modify cluster in `ns-excluded` → no notification

**Test: completions scoped**
- Namespace completion → only suggests `ns-allowed-1` and `ns-allowed-2`
- Cluster name completion → only suggests clusters from allowed namespaces

**Test: empty config = all namespaces**
- Deploy MCP without `MCP_NAMESPACES` → all 3 namespaces visible

## Verification

1. All filtering tests pass
2. No resource from excluded namespace leaks through any tool, watch, or completion
