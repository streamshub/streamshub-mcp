# 5.6 E2E Negative & Edge Case Scenarios

## Context

Test guardrail protections, error handling, and edge cases that could cause issues in production.

## Test Classes

### `RateLimitingE2ETest.java`

**Test: Rate limit triggers on rapid requests**
- Configure low rate limit (e.g., 5 RPM for log category)
- Make 6 rapid `get_kafka_cluster_logs` calls
- Verify 6th call returns rate limit error
- Verify rate limit counter metric incremented

### `ResponseTruncationE2ETest.java`

**Test: Large response truncated**
- Configure low max response bytes (e.g., 1000)
- Invoke tool that returns large response (e.g., `list_kafka_topics` on cluster with many topics)
- Verify response is truncated with truncation indicator
- Verify truncation metric incremented

### `LogRedactionE2ETest.java`

**Test: Sensitive patterns redacted from logs**
- Create Kafka cluster with logs that contain bearer tokens or passwords (e.g., in JAAS config)
- `get_kafka_cluster_logs` → verify sensitive patterns replaced with `[REDACTED]`
- Test custom redaction patterns via configuration

### `PaginationE2ETest.java`

**Test: Topic list pagination**
- Create many topics (e.g., 150)
- `list_kafka_topics` with limit=50, offset=0 → get first 50, `has_more=true`
- `list_kafka_topics` with limit=50, offset=50 → get next 50
- `list_kafka_topics` with limit=50, offset=100 → get last 50, `has_more=false`

### `InvalidInputE2ETest.java`

**Test: Invalid namespace name**
- `get_kafka_cluster(namespace="INVALID_NS!!!")` → validation error

**Test: Non-existent resource**
- `get_kafka_cluster(namespace="valid-ns", clusterName="nonexistent")` → not found error

**Test: Missing required parameter**
- `get_kafka_cluster(namespace="ns")` without `clusterName` → validation error

### `InvalidOidcTokenE2ETest.java` (requires auth enabled)

**Test: Malformed token**
- Send `Authorization: Bearer not-a-jwt` → 401

**Test: Token for wrong audience**
- Generate token with audience `wrong-service` → 401

### `EmptyClusterE2ETest.java`

**Test: Cluster with no node pools**
- Deploy Kafka CR without node pools (edge case)
- `get_kafka_cluster` → returns cluster info, empty node pools
- `diagnose_kafka_cluster` → handles gracefully, notes missing node pools

**Test: Namespace with no Strimzi resources**
- `list_kafka_clusters` in empty namespace → returns empty list, no error

## Verification

1. All negative tests produce expected error responses (not crashes)
2. Guardrail protections work as designed
3. Edge cases handled gracefully
